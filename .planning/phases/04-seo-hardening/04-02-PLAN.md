Create `stella-mattina/src/components/seo/BreadcrumbJsonLd.tsx` as a Server Component (NO 'use client' directive — breadcrumb JSON-LD needs no client JS).

Define a `BreadcrumbItem` type: `{ name: string; url: string }`. Define a `Props` type: `{ items: BreadcrumbItem[] }`.

Export a named function component `BreadcrumbJsonLd({ items }: Props)`. Build the JSON-LD object using `Record<string, unknown>` (consistent with project-wide pattern from Phase 2 — schema-dts has union type issues):

```typescript
const jsonLd: Record<string, unknown> = {
  '@context': 'https://schema.org',
  '@type': 'BreadcrumbList',
  itemListElement: items.map((item, index) => ({
    '@type': 'ListItem',
    position: index + 1,
    name: item.name,
    item: item.url,
  })),
}
```

Return:
```typescript
return (
  <script
    type="application/ld+json"
    dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
  />
)
```

Do NOT add a `'use client'` directive — this is a pure Server Component. Do NOT use schema-dts types for BreadcrumbList — use `Record<string, unknown>` consistent with all existing JSON-LD in the project.

Note: This component must NOT be added to the homepage page.tsx. BreadcrumbList requires at least 2 ListItems to be valid per schema.org — homepage would only have 1 item.
