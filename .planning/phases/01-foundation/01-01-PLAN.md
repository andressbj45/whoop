  <action>
Run from the project root (i.e., the Woop/ directory — where this PLAN.md lives):

STEP 1 — Verify Node.js version (must be 20.9+ for Next.js 16):
```
node --version
```
If below 20.9, stop and report the version. Do not proceed.

STEP 2 — Scaffold the project:
```
npx create-next-app@latest stella-mattina \
  --typescript \
  --tailwind \
  --app \
  --src-dir \
  --import-alias "@/*" \
  --turbopack \
  --yes
```
This creates `stella-mattina/` with: Next.js 16.x, React 19, TypeScript 5.x, Tailwind v4, App Router, src/ directory, @/* alias, Turbopack default.

STEP 3 — shadcn/ui init (MUST run BEFORE adding SM brand tokens to globals.css, or shadcn will overwrite them):
```
cd stella-mattina
npx shadcn@latest init --yes
```
When prompted (if --yes does not fully suppress): select "Default" style, "Slate" base color, CSS variables: yes.

STEP 4 — Add required shadcn components:
```
npx shadcn@latest add button --yes
npx shadcn@latest add navigation-menu --yes
npx shadcn@latest add sheet --yes
```

STEP 5 — Install additional dependencies:
```
npm install next-plausible schema-dts @tailwindcss/typography
```

STEP 6 — Verify tsconfig.json has resolveJsonModule enabled. Open stella-mattina/tsconfig.json and confirm `"resolveJsonModule": true` is present. If missing, add it to compilerOptions. This is required for the content layer to import JSON files.

STEP 7 — Verify globals.css uses Tailwind v4 import style. Open stella-mattina/src/app/globals.css and confirm it starts with `@import "tailwindcss"` (not `@tailwind base;` which is v3 syntax). If you see v3 syntax, replace the three @tailwind directives with `@import "tailwindcss";`.

STEP 8 — Verify the dev server starts:
```
npm run dev &
sleep 5
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
kill %1
```
Expected: HTTP 200. If not 200, check for errors in dev server output.

NOTE: Do NOT add SM brand tokens to globals.css in this task — that happens in Plan 02. This task only scaffolds and verifies the baseline project.
